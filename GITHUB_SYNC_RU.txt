Как работать с GitHub между компьютерами

1. Перед началом работы запусти:
   update.bat

2. Меняешь файлы проекта.

3. После работы запусти:
   save.bat

4. Введи короткое описание изменения, например:
   fix document recognition

5. На другом компьютере снова запусти:
   update.bat

Важно:
- файл .env не отправляй в GitHub;
- ключи API держи только на сервере или локально в .env;
- если GitHub попросит логин, авторизуйся в браузере или через token.
